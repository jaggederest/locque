# Contributing to Locque

Thank you for your interest in contributing to Locque.

## Prerequisites

- **GHC 9.12.2** — the Haskell compiler
- **Cabal 3.16** — the build tool

The CI workflow uses these exact versions. Mismatches may cause build failures.

Install via [GHCup](https://www.haskell.org/ghcup/):

```bash
ghcup install ghc 9.12.2
ghcup install cabal 3.16.1.0
ghcup set ghc 9.12.2
ghcup set cabal 3.16.1.0
```

## Running the test suite

```bash
./tools/ci.sh test
```

This builds the interpreter and runs all `.lq` tests via `smyth test`. To pass
`--verbose` or other flags through to `smyth`:

```bash
./tools/ci.sh test --verbose
./tools/ci.sh test --slow      # report slowest suites
```

To build without running tests:

```bash
./tools/ci.sh build
```

## Code style

- **Haskell**: follow the conventions in `interpreter/`. HLint is enforced in CI.
  Run `hlint interpreter/` locally before submitting.
- **Locque source** (`lib/`, `test/`): follow the surface syntax documented in
  `grammar.md`. Write `.lq` files; `.lqs` mirrors are generated by the converter.
- No new Haskell primitives without explicit instruction — prefer implementing
  functionality in Locque itself.
- Do not delete or skip tests to make them pass.
- Lowercase file paths for all `lib/**/*.lq` and `test/**/*.lq` files (module
  names map to lowercase paths).

## Submitting changes

1. Fork the repository and create a branch from `main`.
2. Make your changes, ensuring all tests pass (`./tools/ci.sh test`).
3. Open a pull request with a clear description of what changed and why.

For questions or design discussions, open an issue first.
